React学习笔记

发表于2022-12-31|更新于2022-12-31
|字数总计:3.3k|阅读时长:14分钟|阅读量:

1.什么是React

用于构建用户界面的JavaScript库，是一个将数据渲染为HTML视图的开源JavaScript库。

2.为什么要学React

  1. 原生JavaScript操作DOM繁琐、效率低。

  2. 使用JavaScript直接操作DOM，浏览器会进行大量的重绘重排。

  3. 原生JavaScript没有组件化编码方案，代码复用率低。

3.React的特点

  1. 采用组件化模式、声明式编码，提高开发效率及组件复用率。

  2. 在React Native中可以使用React语法进行移动端开发。

  3. 使用虚拟DOM+优秀的Diffing算法，尽量减少与真实DOM的交互。

4. 学习React前所需要的JavaScript基础知识

  • 判断this的指向

  • class（类）

  • ES6语法规范

  • npm包管理器

  • 原型、原型链

  • 数组常用方法

  • 模块化

5.JSX语法规则

  1. 定义虚拟DOM时，不要写引号。

  2. 标签中混入JS表达式时要用{}。

  3. 样式的类名指定不要用class，要用className。

  4. 内联样式，要用style={{key:value}}的形式去写。

  5. 虚拟DOM必须只有一个根标签。

  6. 标签必须闭合

  7. 标签首字母

    1. 若小写字母开头，则将该标签转为html同名元素，若html中吾该标签对对应的同名元素，则报错。

    2. 若大写字母开头，react就去渲染对应的组件，若组件没有定义，则报错。

  8. 动态输出一系列li时，{}内输入的是表达式

    一定注意区分【js语句（代码）】和【js表达式】

    1. 表达式：一个表达式会产生一个值，可以放在任何一个需要值得地方

      • a

      • a+b

      • demo(1)

      • arr.map()

      • function test () {}

    2. 语句（代码）

      • if () {}

      • for () {}

      • switch () {case:xxx}

6.React面向组件编程

类class知识的回顾

  1. 类中的构造器不是必须写的，要对实例进行一些初始化操作，如：填写属性时才写；

  2. 如果A类继承B类，且A类上有构造器，那么A类构造器中的super是必须调用的；

  3. 类中所定义的方法，都是放在了类的原型对象上，供实例去使用

7.创建类式组件

使用class来创建组件，注意要继承React.Component

类式组件内的三大属性

  1. State

    注意点：

    1. 要想给State传值，就必须在初始化时使用构造器赋值 constructor()，同时super()是必须要写的

    2. 说是传值，其实是传key:value对象

    3. 在进行绑定事件的时候，React官方推荐使用直接在标签上调用方法。React重写了HTML所有的事件代码，全部改成驼峰式命名。调用绑定的事件不是原生js的写法 "demo()"，而是 {demo}。因为加上括号后会把方法的返回值（undefined）赋给click事件

    4. 改变state，必须使用setState()，在构造器内使用.bind(this)可获得实例内部的state，从而取反，否则会显示undefined。

  2. State简写模式

    其实就是去除构造器，把需要初始化的东西直接通过赋值完成state = { isHot: false }，下方的changeWeather方法也是用赋值的方法挂到原型对象上，避免了前文在构造器内.bind()

    总结：

    1. state是组件对象最重要的属性，值是对象（可以包含多个key-value的组合）

    2. 组件被称为“状态机”，通过更新组件的state来更新对应的页面显示（重新渲染组件）

    3. 组件中render方法中的this为组件实例对象

    4. 组件自定义的方法中this为undefined，如何解决？

      • 强制绑定this：通过函数对象的bind()

      • 箭头函数

    5. 状态数据，不能直接修改或更新，必须使用setState()

  3. Props

    使用{...p}，可以批量将参数传入类内，key:value

    1. Props简写方式

      上述代码的Person.propTypes 和Person.defaultProps最好能放回类本身，使用static关键字即可把propTypes和defaultProps成为类的属性。

    2. 函数组件使用props

      组件实例的三大组件，函数组件只能玩props，其他的两个都不行。

      propTypes和defaultPros在函数体外定义即可

    3. 字符形式的refs(不推荐使用)

      refs是类中的一个属性，主要的作用是标识标签，类似document.getElementByID()，拿到具体的真实的标签，再对它进行操作

    4. 回调函数的refs

      把ref="inputLeft"改成ref={c =>this.input1=c}。这是一个回调函数，this是指实例对象，相当于把input1这个标签挂到了实例对象上。

      上述例子为一个内联函数，实际生产环境下是更推荐使用，产生的问题是无关紧要的。如果要较真，那就是如果使用了内联函数，React会在refs变更的时候，调用内联函数2次，第一次的this为NULL。原因是：更改了refs，render就会重新渲染一次，之前的内联函数就会被清除，所以第一次调用的this为NULL。

    5. createRef的使用

      使用React官方推荐创建Refs的API——React.createRef()，可以直接获取到对应标签的内容，但是稍许麻烦，要创建多少个ref就要多少个容器（较麻烦，但是React官方最推荐使用）

      上述代码中的第二个<input>，里面的ref可以省略，因为后面的onBlue在React调用的时候会同步传入event到方法内，该方法已经能够获取到了当前节点。

    6. 非受控组件

      将需要收集的信息用ref进行收集，在提交表单时才把信息取出来再提交

    7. 受控组件

      使用onChange作为触发事件，只要输入框有修改，信息则传入state，提交表单时从state里取信息出来
